# AP6275S — WiFi 6 + Bluetooth 5.3 SiP module

**Designator:** U2

**Wiring targets:** U1/RK3576

**Vendor:** AMPAK Technology Inc. **Datasheet:** `cradle_sidecar/data/datasheets/AP6275S Documentation.pdf` (includes EVB user manual, pages 55-63). **Structural index:** `cradle_sidecar/data/datasheets/AP6275S.index.md` (auto-generated page pointers — open on demand, not pre-read).

This file holds two different kinds of fact, kept in separate columns so they never get confused: what the **vendor** documents (stable, reusable, part-specific), and what **Cradle** actually does with each pin (changes as design decisions get made, cites `decisions-log.md`). Vendor facts are tagged `[VERIFIED — source]` or `[UNVERIFIED — source]` per the provenance convention in `cradle_sidecar/co-design-workflow.md`.

## High-level summary

- Broadcom BCM43752-based combo module: WiFi 6 (802.11ax) 2x2 MIMO + Bluetooth 5.3, SMD-50P, 15x13mm.
- Host interfaces: SDIO 3.0 (4-bit) for WiFi, UART for Bluetooth HCI. PCM interface exists but Cradle doesn't route it (see pin 44-47 below).
- No integrated antenna — `WL_ANT0`/`WL_ANT1` are bare RF I/O pins requiring external antennas + matching.
- Needs two external clocks not shared with the host SoC: 37.4MHz (`XTAL_IN`/`XTAL_OUT`) and 32.768kHz (`LPO`).
- Has two internal switching regulators (digital core + analog core) whose inductor must be placed on the **host board**, not integrated in the module — a real gap in AMPAK's own datasheet, see Known Gaps.

## Known gaps (things AMPAK's datasheet does not specify)

- **No inductor/cap value for the `CBUCK_0P9`/`CSR_VLX` and `ASR_VLX`/`ABUCK_1P12` loops.** Confirmed absent in both the AP6275S datasheet and its PCIe-variant sibling AP6275P (checked both, same gap). Likely because the switching frequency and loop compensation are proprietary to Broadcom's die. Resolved via two independent production schematics — see External Validation below — rather than a first-principles calc, since neither the switching frequency nor per-rail load current are published anywhere found.
- **No stated value for a series resistor on `SDIO_DATA_CLK`.** AMPAK's EVB manual only tables pull-up values for CMD/D0-D3, not CLK, and never states a value for the `R113` footprint visibly placed at the CLK pad in EVB photos.
- Pin 24 is documented as NC in this datasheet, but a third-party schematic symbol (shared across the AP62x2 family) labels the same physical pin `SDIO_VSEL`/`PCIE_WAKEn` for other family members — see External Validation.

## External validation (secondary sources, not the primary datasheet)

Per `cradle_sidecar/co-design-workflow.md` convention: facts extracted here are cited, source files are not persisted in-repo (one carries a confidentiality watermark).

- **Geniatech CBD-3588** (RK3588 SOM, sheet "62.WIFI/BT-SDIO_AP6275S", file.geniatech.com, accessed 2026-07-07): `CBUCK`/`ABUCK` inductors = 4.7uH (3030 package, "1A_DCR<=80mOhm" explicitly annotated), output caps 4.7uF. SDIO pull-ups populated per AMPAK's own EVB table (30kOhm, CMD+D0-D3, not CLK). Antenna: single 10pF series DC-block per path, pi-network shunt legs present but DNP. `LPO_IN` fed from a shared system 32kHz clock, not a dedicated oscillator (Cradle diverges here intentionally). `[VERIFIED — schematic rendered+zoomed, not text-extracted — this PDF's text layer scrambles badly]`
- **Geniatech OSM_BB / SOM-iMX8MM-OSM** (i.MX8MM SOM, sheet "P09 WIFI6/BT (AP6275S)", file.geniatech.com, accessed 2026-07-08): `CBUCK`/`ABUCK` inductors = **2.2uH** (3030 package), output caps 4.7uF — different value than CBD-3588 for the identical role, confirming this is a tolerant loop (2.2-4.7uH range), not a razor-tuned single value. `R76` = 100Ohm populated series resistor between the 32kHz oscillator output and `LPO_IN`. `R73`(120Ohm)+`C71`(20pF) form a ~66MHz-cutoff RC filter directly at the `LPO_IN` pin — well above 32.768kHz, so it's noise/EMI suppression, not signal shaping. All 6 SDIO lines (including CLK) get both a 30kOhm pull-up and a 0Ohm series-R footprint in this design, unlike CBD-3588 or AMPAK's own EVB table. `[VERIFIED — schematic rendered+zoomed]`
- **AP6275P datasheet** (SparkLAN/varitec.co.jp, ver1.2, 2023-04-20): checked for a CBUCK/ABUCK reference circuit the AP6275S datasheet might be missing — confirms the same gap exists in this sibling part's datasheet too (pin table only, no application circuit). `[VERIFIED — direct PDF text read]`
